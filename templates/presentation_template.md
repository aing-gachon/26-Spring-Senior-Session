# 발표 슬라이드 작성 가이드

## 팁

- 모델의 성능보다 어떤 고민을 했고, 어떤 에러를 어떻게 해결했는지 과정을 상세히 담으면 좋습니다.
- 제공된 6개 목차를 기반으로 논리적인 흐름을 유지해 주세요.
- 제공된 템플릿은 예시일 뿐입니다. 사본을 만든 후 프로젝트 특성에 맞춰 슬라이드를 추가/삭제/수정해도 무방합니다.
- 성능 비교 그래프, 모델 다이어그램, 데모 스크린샷은 명확하게 제시해야 합니다.

## 목차별 슬라이드 구성

| 섹션 | 주요 포함 내용 |
|------|----------------|
| 01. Background | 주제 선정 이유, 레퍼런스 레포 링크, 핵심 목표 3가지 |
| 02. Env & Structure | 실행 환경(Colab 등), 코드 모듈 구조, 데이터 특성 |
| 03. Troubleshooting | 데이터 연동 시 수정한 부분, 발생한 에러 및 해결 과정 |
| 04. Optimization | 튜닝 결과, 모델 구조 변경, 시도했던 기술적 실험들 |
| 05. Result & Demo | 최종 성능 지표, 데모 영상/이미지, 블로그 기록물 |
| 06. Retrospective | 프로젝트 한계점, 배운 점, 향후 발전 방향 |

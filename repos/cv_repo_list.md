# CV 레포 추천

## 선정 기준
- 학습/추론이 재현 가능하며 문서화가 잘 되어 있을 것
- 데이터/모델 구조가 명확해 수정하기 쉬울 것
- 실험 로그와 결과가 비교 가능하게 정리되어 있을 것

## 주의 사항
- 해당 레포는 추천 자료일 뿐, 반드시 이 중에서 선택해야 하는 것은 아닙니다. 다른 레포를 찾아보거나, 논문 구현 레포를 직접 찾아보는 것도 좋은 방법입니다.
- 운영진은 추천 레포 리스트만 제공하며, 레포 선정과 실험 과정에서 발생하는 문제에 대해서는 지원하지 않습니다. 팀원들과 협력하여 문제를 해결해 나가는 것도 중요한 과정입니다.

## 후보 레포 1 : Ultralytics (YOLO Series)
- 링크: https://github.com/ultralytics/ultralytics?tab=readme-ov-file
- 핵심 내용
    - YOLOv8 모델을 활용한 객체 탐지, 분류, 세분화 프로젝트
    - 단 몇 줄의 코드로 커스텀 데이터 학습 가능
    - ONNX/TensorRT 등 배포 모듈 지원
- 장점 
    - 풍부한 문서화와 튜토리얼
    - 다양한 모델과 데이터셋 지원
    - 다양한 Tasks(탐지, 분류, 세분화) 지원
- 주의사항
    - GPL-3.0 라이선스이므로, 상용화 시 코드를 공개해야 할 수 있음
    - 사용이 쉬워 초보자에게 적합하지만, 모델 구조를 깊이 이해하기에는 다소 추상적일 수 있음
    - 대규모 모델을 학습할 때는 GPU 자원이 많이 필요할 수 있음

## 후보 레포 2 : OpenMMLab Detection (MMDetection)
- 링크: https://github.com/open-mmlab/mmdetection
- 핵심 내용
    - Faster R-CNN부터 최신 Transformer 기반 모델까지 수백 개의 알고리즘을 모듈 단위로 쪼개 놓은 거대 라이브러리
    - Config 시스템을 통해 Backbone, Neck, Head를 레고 조립하듯 바꿔 끼우며 실험할 수 있음
- 장점
    - 데이터 로더 - 모델 - 학습 루프가 어떻게 객체지향적으로 설계되어야 하는지 배우기에 가장 좋은 교보재
    - 거의 모든 SOTA 모델이 구현되어 있어, 논문 스터디 후 바로 코드를 확인해보기 좋음
    - 본인의 아이디어를 추가하여 새로운 모듈을 만들기 용이함
- 주의사항
    - Config 파일 상속 구조가 복잡하여 초반에 환경 설정과 데이터 포맷을 맞추는 데 애를 먹을 수 있음
    - 최신 Transformer 계열 모델은 무거운 편이라, 코랩 무료 버전이나 저사양 로컬 환경에서는 학습이 어려울 수 있음

## 후보 레포 3 : Intel Anomalib (Anomaly Detection)
- 링크: https://github.com/openvinotoolkit/anomalib
- 핵심 내용
    - 정상 데이터만으로 학습하여 비정상을 찾아내는 이상 탐지 라이브러리
    - 최신 비지도 학습 알고리즘을 제공하며, 불량 부위를 히트맵으로 시각화해주는 기능 포함
- 장점
    - 스마트 팩토리나 품질 검사 등 실제 산업계 수요가 매우 높은 분야의 대표적인 문제를 다룸
    - 정상 데이터만 있으면 학습이 가능하므로, 데이터 수집과 라벨링 부담이 적음
    - Pytorch Lightning 기반으로 설계되어 있어, 학습 루프와 모델 구조가 깔끔하게 분리되어 있음
- 주의사항
    - Accuracy가 아닌 AUROC, F1-score 등의 지표가 중요하므로, 통계적 평가 방법에 대한 사전 지식이 필요할 수 있음
    - 폴더 구조(Good/Bad/Mask)를 엄격하게 지켜야 하므로 초기 데이터 세팅에 주의가 필요함
    - 최신 알고리즘은 모델이 무거운 편이라, GPU 자원이 제한된 환경에서는 학습이 어려울 수 있음

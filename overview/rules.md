# 출석 및 참여 규칙

시니어 트랙에서는 프로젝트 진행 속도, 주제 선정 등 모든 것을 팀이 자율적으로 결정하지만, 출석과 참여는 필수입니다. 아래 규칙을 참고해주세요.

## 기본 원칙
팀 프로젝트로 진행되는 만큼, **출석과 참여도가 굉장히 중요**합니다.

## 출석 관리
- 출석 원칙은 **팀장 전결제**로 하며, 판단 권한은 전적으로 팀장에게 있습니다.
- 팀장은 매주 활동 결과물과 출석 명단을 제출해야 합니다.
- 부당한 처리 발생 시 **디스코드 신문고**로 제보해주세요.

## 결석 처리
- **무단 결석 1회 발생 시 활동 중지 처리**됩니다.
- 해당 주차 결과물 미제출 시 무단 결석으로 간주됩니다.

## 공결 처리
- 공결 시 몸이 못 오면 **기여로 인정되는 활동**을 제출해야 합니다.

### 8주 코스 (1개 프로젝트)
| 주차 | 대체 과제 예시 |
|------|----------------|
| 1주차: 기획 및 탐색 | 팀에서 정한 주제에 대한 배경 조사 및 관련 논문/자료 정리, 유사 프로젝트 벤치마크 분석, Github 레포지토리 분석 및 실행 |
| 2주차: 탐색 심화 | 데이터셋 분석 문서 작성, 베이스라인 코드 분석 정리, 이슈 및 개선점 리스트업 |
| 3주차: 튜닝 | 데이터 전처리 코드 작성/수정, 실험 설정 문서화, 코드 리뷰 및 피드백 |
| 4주차: 튜닝/에러 해결 | 에러 디버깅 및 해결 방안 문서화, 재현성 체크리스트 작성, 학습 파이프라인 코드 개선 |
| 5주차: 최적화 | 하이퍼파라미터 실험 결과 정리, 모델 개선 아이디어 및 구현, 성능 비교 표/그래프 작성 |
| 6주차: 최적화/확장 | 추가 실험 설계 및 실행, 트레이드오프 분석 문서 작성, 코드 리팩토링 및 최적화 |
| 7주차: 정리 | 결과 시각화 및 문서 작성, 블로그/Velog 작성, README 작성 및 정리 |
| 8주차: 발표 | 발표 자료 제작, 데모 준비 및 테스트, 최종 문서 정리 |

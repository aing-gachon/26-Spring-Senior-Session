# 주차별 일정

## 프로젝트 1개 진행 시 (8주)
### 1주차: 기획 및 탐색
#### 활동 내용
- 주제 선정 및 목표 정의
- 참고/유사 프로젝트 조사
- GitHub 레포지토리 선정 및 실행
- 코드 구조 파악 및 실행 로그 작성
#### 결과물
- 선정된 깃허브 링크
- 실행 성공 스크린샷

### 2주차: 탐색 심화
#### 활동 내용
- 데이터 구조 및 라벨 확인
- 베이스라인 재현 및 성능 확인
- 주요 이슈 리스트업
#### 결과물
- 베이스라인 재현 로그
- 이슈 리스트

### 3주차: 튜닝
#### 활동 내용
- 내 데이터셋 적용 또는 데이터 로더 수정
- 전처리 파이프라인 정리
- 실험 세팅 고정 및 기록 양식 정비
#### 결과물
- 내 데이터 적용 후 학습 로그
- 전처리/실험 설정 문서

### 4주차: 튜닝/에러 해결
#### 활동 내용
- 학습 파이프라인 안정화
- 주요 에러 해결 및 재현성 확보
- 실험 실패 원인 분석
#### 결과물
- 에러 해결 로그
- 재현 가능한 실행 체크리스트

### 5주차: 최적화
#### 활동 내용
- 모델 구조 개선 또는 하이퍼파라미터 튜닝
- 성능 비교 실험 및 결과 정리
#### 결과물
- 개선된 모델의 학습 로그 및 성능 비교
- 튜닝 결과 요약 표

### 6주차: 최적화/확장
#### 활동 내용
- 추가 실험(어블레이션, 데이터 확장 등)
- 성능/속도/자원 트레이드오프 분석
#### 결과물
- 추가 실험 결과 정리
- 트레이드오프 분석 요약

### 7주차: 정리
#### 활동 내용
- 결과 정리 및 시각화
- 블로그/Velog 기록
- 데모/서비스 형태 연결(웹, API, 노트북 등)
#### 결과물
- 그래프/표가 포함된 결과 정리 문서
- 데모 링크 또는 실행 가능한 노트북
- 블로그/Velog 링크

### 8주차: 발표
#### 활동 내용
- 최종 결과물 정리(README, 로그, 데모)
- 발표 자료 제작 및 리허설
- 피드백 수집 및 개선점 정리
#### 결과물
- 최종 README.md
- 발표 자료(슬라이드, 데모 링크 등)
